Introducing Synchronous Dispatcher

The Easiest And Fastest Event Framework for React Ever!

Posted by Heikki Kupiainen/Oppikone on 18.07.2017 09:37:36

Synchronous Dispatcher Is Simpler, Cleanear And Better

In this article I will introduce how to use the super lightweight yet extremely powerful and intuitive Synchronous Dispatcher event framework that I have developed just for you, dear Reader, to help you create fantastic software in a painless way.

Synchronous Dispatcher is a wonderful little framework - or should I call it rather a design pattern - that I designed already in 2010 for ActionScript programming language. Those times ActionScript (used by Flash-based web apps) already had the maturity that JavaScript-based techniques are now slowly catching up. Some who think that React is something new may be surprised to hear that React is essentially only a remake or at best a reinvention of Adobe Flex (later called Apache Flex) framework that was introduced already in 2004 - it essentially introduced a virtual DOM model: you could create your own Flex components that you would then use as construction blocks inside your HTML-like templates. Adobe Flex really already had everything that React has today!

Mate Framework introduced in 2010, event framework for Adobe Flex, was exactly the same that "Redux" is today for "React" - but Mate Framework had exactly the same weakness that Redux has today. It is unnecessarily complex! But Redux is even worse: the way how it handles events using endless lists of switch case structures is in my opinion quite awful - don't you think we could have something sleaker in 2017 than dozens of times if else if else if else if else that Redux-style switch-casing essentially is? Moreover, Redux sort-of forces the developer to divide components into "container" components and "representation" components. While this makes sense in theory, in practice the separation is quite articifial and forces the developers into rigid programming style that will yield counterproductive results - you will get awkward complexity instead of clarity that was perhaps the initial intention!

I bet that you may have already had this creepy feeling with Redux: do things really need to be this complicated? I had this same feeling in 2010 when I had to take over a project that was using Mate. Therefore I created my own event framework - super simple, fast to implement, a lot cleaner, more intuitive and fun to use! Later I ported that for JavaScript and now that I see that the new JavaScript generation is repeating with Redux the same mistakes that the previous did with Mate for Flex - it is time to show how to use Synchronous Dispatcher together with React. Let's make coding fun again!

Using Synchronous Dispatcher

Using Synchronous Dispatcher is simple: when you want make a React component to listen for an event you jast add a handle to the constructor of that component, for instance:

handle('CAR-MODEL-SELECTED', this.onCarModelSelected)

And wherever you want to dispatch the event, you just call dispatch method:

dispatch('CAR-MODEL-SELECTED', carModelId)
